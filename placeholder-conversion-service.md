# 服务端占位符规则

本文档描述服务端的占位符使用规则。
## 推荐语法

占位符的通用格式如下：

```text
%[参数顺序][标志][宽度][.精度]类型
```

指定参数顺序时使用方括号：

```text
%[参数顺序]类型
```

| 占位符 | 含义 |
|---|---|
| `%s` | 字符串 |
| `%d` | 十进制整数 |
| `%f` | 浮点数 |
| `%t` | 布尔值 |
| `%c` | Unicode 码点表示的字符 |
| `%%` | 百分号本身，不消耗参数 |
| `%[2]s` | 第 2 个字符串参数 |
| `%02d` | 整数最小宽度为 2，不足时补 0 |
| `%.2f` | 浮点数保留 2 位小数 |
| `%[2].2f` | 第 2 个浮点数参数，保留 2 位小数 |

## 基础占位符

| 占位符 | 含义 |
|---|---|
| `%s` | 字符串 |
| `%d` | 十进制整数 |
| `%f` | 浮点数 |
| `%t` | 布尔值 |
| `%c` | Unicode 码点表示的字符 |
| `%%` | 百分号本身 |

> 使用 `%t` 格式化布尔值。Go 中的 `%b` 用于将整数格式化为二进制，不应作为布尔值占位符。

## 带顺序的占位符

使用 `%[n]类型` 指定第 `n` 个参数，参数顺序从 `1` 开始。

| 占位符 | 含义 |
|---|---|
| `%[1]s` | 第 1 个字符串参数 |
| `%[2]s` | 第 2 个字符串参数 |
| `%[12]s` | 第 12 个字符串参数 |
| `%[1]d` | 第 1 个整数参数 |
| `%[2]d` | 第 2 个整数参数 |
| `%[1]f` | 第 1 个浮点数参数 |
| `%[2]f` | 第 2 个浮点数参数 |
| `%[1]t` | 第 1 个布尔参数 |
| `%[2]t` | 第 2 个布尔参数 |
| `%[1]c` | 第 1 个字符参数 |
| `%[2]c` | 第 2 个字符参数 |

不使用 Android / iOS 的 `%2$s` 参数顺序语法，对应的 Go 原生占位符是 `%[2]s`。

## 宽度与补零

| 占位符 | 含义 |
|---|---|
| `%02d` | 整数最小宽度为 2，不足时补 0 |
| `%03d` | 整数最小宽度为 3，不足时补 0 |
| `%04d` | 整数最小宽度为 4，不足时补 0 |

## 浮点数精度

| 占位符 | 含义 |
|---|---|
| `%.1f` | 浮点数保留 1 位小数 |
| `%.2f` | 浮点数保留 2 位小数 |
| `%.3f` | 浮点数保留 3 位小数 |

## 带顺序的浮点数精度

| 占位符 | 含义 |
|---|---|
| `%[1].1f` | 第 1 个浮点数参数，保留 1 位小数 |
| `%[1].2f` | 第 1 个浮点数参数，保留 2 位小数 |
| `%[1].3f` | 第 1 个浮点数参数，保留 3 位小数 |
| `%[2].1f` | 第 2 个浮点数参数，保留 1 位小数 |
| `%[2].2f` | 第 2 个浮点数参数，保留 2 位小数 |
| `%[2].3f` | 第 2 个浮点数参数，保留 3 位小数 |

## 服务端使用方式

翻译文本可以直接作为 Go 格式化字符串使用：

```go
message := fmt.Sprintf("球员 %[1]s 获得了 %[2]d 分", playerName, score)
```

使用时应遵循以下规则：

- 占位符类型必须与参数类型匹配，例如字符串使用 `%s`，布尔值使用 `%t`。
- 翻译文本可能调整参数顺序时，应使用带顺序的占位符。
- 输出百分号时必须保留 `%%`。
- 翻译文本应作为受控的格式化模板使用，不应将不可信输入直接作为格式化字符串。

## 语法规则汇总

| 规则 | 占位符 |
|---|---|
| 字符串 | `%s` |
| 布尔类型 | `%t` |
| 指定参数顺序 | `%[2]s` |
| 指定宽度并补零 | `%02d` |
| 指定浮点数精度 | `%.2f` |
| 同时指定参数顺序和精度 | `%[2].2f` |
| 输出百分号 | `%%` |
